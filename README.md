#### 预训练数据处理---注意数据配比【可参考ChatHome论文】
    - 采用直接裁剪的方式
     0.按照4096进行训练，所有数据直接进行拼接，然后按照4096进行切片
     1.对于较短的文本，比如问答、选择题这类数据，采用拼接的方式；
     2.预训练的数据格式上的处理，每条后面加结束符，比如问答数据：q+'\n'+a+'</s>'；对话数据：患者：xx\n医生:yy</s>
     3.预训练的文本，拼接成1024或2048..，形如:<s>text</s>形式进行模型
     4.注意数据多样性，真的狠重要啊！！！
     5.一定要去重！！！

    - 对于自身显存受限，可以选用基座模型长度为4096或更长， 然后继续预训练长度设为2048
    - 微调时，再将长度切换为4096，我在qwen7B，尝试过，测试过一些例子，4096长度推理没有问题。

#### 微调数据收集---注意数据配比【可参考ChatHome、llama2官方论文】
    - 通用数据收集
      0. 首选GPT4产生的数据，相对质量高，目前开源的数据主要有 sharegpt_zh、alpaca-zh、wizard-70k等
      1. 其他一些开源数据，有人工整理和gpt3.5产生的一些数据
    - 领域类数据
      0. 其实目前开源的中文医疗数据，并没有发现质量比较好的指令集，相对不错的有一些比赛的数据。
      1. 使用书籍、指南等材料，进行问答或对话数据的抽取
      2. 人工构建
    - 指令需要进行去重，指令的单一或重复对模型的泛化性影响很大
     
#### 微调数据处理
     0.微调数据中单论问答和多轮问答，都规定了角色信息，输入端id: User, 模型输出端id:Assistant;
     1.所有文本，拼接都固定长度，如2048或4096，形如：User:q1</s>\n Assistant:a1</s>\n User:q2</s>\n Assistant:a2....;
     2.拼接时，单轮进行混合拼接； 多轮数据进行混合拼接
     3.微调时，将label中输入的部分进行屏蔽，其label值设为-100(tokenize和注释掉的那部分)
     4.增加Lora指令微调方式，不过目前仅支持单卡微调，多卡出现了问题，后续解决了再更。
     
     数据形式：【注意角色的冒号后面空一格，因为有些符号跟冒号在一起会被识别成一个】
     User: 关于右哌甲酯和为什么它是处方药</s>\n Assistant: 右哌甲酯是一种用于治疗儿童和成人注意缺陷多动障碍（ADHD）的药物。它的工作原理是增加大脑中某些化学物质的水平，这些化学物质有助于集中注意力。</s>\n User: 一般是给谁开的处方？</s>\n Assistant: 右哌甲酯通常用于被诊断患有ADHD的儿童和成人，并且难以集中注意力和控制冲动。</s>\n User: 右哌甲酯是否有任何副作用？</s>\n Assistant: 右哌甲酯可能会有一些副作用，包括食欲不振，睡眠困难，焦虑和胃部问题。重要的是在开始用药前与医生讨论任何可能的副作用。</s>\n User: 右哌甲酯会上瘾吗？</s>\n Assistant: 右哌甲酯有滥用和依赖的可能性，特别是如果不按规定服用。在开始用药前，与医生讨论有关成瘾或依赖的任何问题是很重要的。</s>
